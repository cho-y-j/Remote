1. 기본 개념

DeskOn은 두 기기를 연결합니다. 한쪽은 제어받는 기기(보여주는 쪽), 다른 쪽은 제어하는 기기(보는 쪽).

  • ID: 9자리 숫자. 각 기기 고유 번호.
  • 비밀번호: 접속할 때 한 번 입력. 매번 새로 생기는 1회용 또는 영구 설정 가능.

제어하는 사람이 상대 기기의 ID와 비밀번호를 알면 접속해서 화면을 볼 수 있어요.

서버 주소와 인증키는 앱에 미리 박혀 있어 따로 설정할 필요 없습니다.

2. Windows 설치/실행

  1. 다운로드 페이지에서 .exe 또는 .msi 다운로드.
  2. 다운로드한 파일 더블클릭 → 설치 진행.
    Windows SmartScreen 경고가 뜨면 “추가 정보” → “실행”을 누르세요. 자체 서명이 없어 처음 실행 시 경고가 뜹니다.
  3. 설치 후 바탕화면 아이콘으로 실행. 본인의 ID(9자리 숫자)가 표시됩니다.
  4. 이 ID와 화면에 표시된 비밀번호를 상대에게 알려주면 끝.

3. macOS 설치/실행

  1. 다운로드 페이지에서 본인 칩에 맞는 .dmg 받기.
    • M1/M2/M3 등 Apple Silicon → aarch64
    • 2020년 이전 Intel Mac → x86_64
  2. 받은 dmg 더블클릭 → 앱을 Applications 폴더로 끌어다 놓기.
  3. 처음 실행 시 “확인되지 않은 개발자” 경고가 뜨면:
    • 시스템 설정 → 개인정보 보호 및 보안 → 아래쪽 “여전히 열기”
  4. 접근 권한 요청 (화면 기록, 손쉬운 사용 등)을 모두 허용. 허용 안 하면 원격 제어가 안 됩니다.

4. Android 설치/실행

  1. 다운로드 페이지에서 .apk 받기.
  2. 설치 시 “출처를 알 수 없는 앱” 차단이 뜨면 → 설정에서 해당 브라우저 앱 허용.
  3. 앱 실행 후 “화면 캡처” 권한과 “접근성” 권한 모두 허용.
    Android는 “화면 공유”까지는 가능하지만 “원격 제어”(터치 입력)는 일부 기기에서 제한됩니다. 삼성 등 일부 제조사는 추가 권한 설정이 필요합니다.
  4. 화면에 ID가 표시되면 준비 완료.

5. Linux 설치/실행

Ubuntu / Debian

sudo dpkg -i rustdesk-1.4.5-x86_64.deb
sudo apt-get install -f -y   # 의존성 자동 설치

설치 후 시스템 트레이 또는 “활동” 화면에서 RustDesk 실행.

Fedora / RHEL

sudo dnf install rustdesk-1.4.5-x86_64.rpm

AppImage (배포판 무관)

chmod +x rustdesk-1.4.5-x86_64.AppImage
./rustdesk-1.4.5-x86_64.AppImage
Linux 데스크톱 환경에 따라 화면 캡처 권한 설정이 다릅니다. Wayland를 쓰는 경우 일부 기능이 X11보다 제한될 수 있어요.

6. 원격 접속하는 법

  1. 제어하려는 기기에서 앱 실행.
  2. 가운데 입력창에 상대방 ID(9자리) 입력 → “연결”.
  3. 비밀번호 입력 창이 뜨면 상대방이 알려준 비밀번호 입력.
  4. 연결 성공 → 상대방 화면이 보이고 마우스/키보드로 제어 가능.
비밀번호는 접속할 때마다 상대방 화면에 표시되는 일회용입니다(영구 비밀번호를 따로 설정하지 않은 경우).

7. 영구 비밀번호 설정

매번 비밀번호를 알려주기 번거로우면 영구 비밀번호를 설정하세요. 자기 기기의 모든 곳에서:

  1. 앱 실행 → 좌측의 ID 옆 비밀번호 변경 (또는 설정 메뉴).
  2. “영구 비밀번호 사용” 선택 → 원하는 비밀번호 입력.
  3. 이후부터 그 비밀번호로 어디서든 접속 가능.
영구 비밀번호는 한 번 새면 누구나 접속 가능합니다. 길고 복잡하게 설정하세요.

8. 자주 묻는 질문

Q. 접속이 안 돼요.

먼저 양쪽 기기 모두 인터넷에 연결되어 있는지 확인. 둘 다 화면에 ID가 떠 있으면 정상이에요. 그래도 안 되면 방화벽이나 회사망/학교망에서 차단된 경우가 많습니다.

Q. 화면이 안 보여요(검은 화면).

제어받는 쪽 기기에서 “화면 기록” 권한이 빠진 경우가 대부분. macOS/Android는 권한 재확인. Linux는 Wayland를 X11로 바꿔보세요.

Q. 연결은 되는데 마우스/키보드가 안 먹어요.

제어받는 쪽 “손쉬운 사용 / 접근성” 권한이 빠졌을 가능성. 권한을 부여한 뒤 앱을 다시 실행하세요.

Q. 안전한가요?

모든 통신은 종단간 암호화(end-to-end encryption)됩니다. 자체 서버를 운영하므로 외부 업체로 데이터가 흘러가지 않아요. 단, 영구 비밀번호 관리는 본인 책임입니다.

Q. iPhone에서도 되나요?

iOS 앱은 준비 중입니다. 현재는 다른 OS에서 iPhone 화면을 “보는 것”만 일부 가능.
